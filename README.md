# TG-WS Proxy Magisk
Rust-реализация MTProto-прокси через WebSocket для Android (Magisk/KernelSU/APatch). Предназначено для обхода ограничений, когда стандартный TCP трафик Telegram блокируется или замедляется.
## Особенности
* Rust-core: Высокая производительность и низкое потребление ресурсов.
* WebSocket: Маскировка трафика под стандартный HTTPS.
* Action Button: Запуск и мгновенное получение ссылки кнопкой в менеджере модулей.
* Config-driven: Гибкая настройка через текстовый файл.
* Auto-connect: Автоматическое открытие Telegram с готовой кнопкой подключения.
* CD_BYPASS: Система автоматического подбора оптимального домена Cloudflare.
## Настройка
Конфиг находится здесь: /data/adb/modules/tg_ws_proxy_f1ndle/config.conf

| Параметр | Описание |
| :--- | :--- |
| PORT | Локальный порт (по умолчанию 443) |
| HOST | Адрес (по умолчанию 127.0.0.1) |
| SECRET | 32 hex символа. Если пусто - генерируется автоматически. |
| CF_DOMAIN | Домен Cloudflare. Если пусто и CD_BYPASS=ON - выберет сам. |
| FAKE_TLS | Домен для маскировки TLS трафика. (по умолчанию : www.mozilla.org |
| AUTOSTART | ON - запуск при загрузке системы. |
| CD_BYPASS | ON - автоматический выбор лучшего домена из списка. |

## Использование
1. Открой Magisk / KernelSU / APatch.
2. Найди модуль в списке и нажми кнопку Action.
3. Прокси запустится, откроется Telegram с предложением добавить прокси.
4. Повторное нажатие кнопки Action останавливает прокси.
## Логи
Если прокси не запускается, проверьте файл:
/data/adb/modules/tg_ws_proxy_f1ndle/proxy.log
