# TG-WS Proxy Magisk

Rust-реализация MTProto-прокси через WebSocket для Android (Magisk/KernelSU/APatch). Идеально для обхода блокировок, когда обычный TCP до Telegram заблокирован.

## Особенности
* Rust-core: Работает быстро и потребляет мало ресурсов.
* WebSocket: Маскирует трафик под стандартный HTTPS.
* Smart Action: Запуск и мгновенное подключение кнопкой в менеджере модулей.
* Config-driven: Настройка через текстовый файл.
* Auto-connect: Автоматически открывает Telegram с готовой кнопкой подключения.

## Настройка
Конфиг находится здесь: `/data/adb/modules/tg_ws_proxy_magisk/config.conf`

| Параметр | Описание |
| :--- | :--- |
| PORT | Локальный порт (по умолчанию 1443) |
| HOST | Адрес (по умолчанию 127.0.0.1) |
| SECRET | 32 hex символа (без dd в начале). Если пусто - будет рандом. |
| CF_DOMAIN | Домен Cloudflare для обхода жестких блоков. |
| AUTOSTART | ON - запускать при загрузке системы (нужен статический SECRET). |

## Использование
1. Открой Magisk / KSU.
2. Нажми кнопку Action (или Настроить) у модуля.
3. Прокси запустится, и тебя перекинет в Telegram с кнопкой Подключить.
4. Нажми кнопку еще раз для остановки процесса.

## Аргументы (EXTRA_ARGS)
Эти флаги можно вписать в EXTRA_ARGS внутри конфига:

| Аргумент | Описание |
| :--- | :--- |
| --dc-ip <DC:IP> | Свой IP для дата-центра (напр. 2:149.154.167.220) |
| --pool-size <N> | Количество соединений в пуле (по умолчанию 4) |
| -v | Включить DEBUG логи для отладки |
| --danger-accept-invalid-certs | Пропустить проверку TLS сертификатов |
| --ws-connect-timeout <SEC> | Таймаут соединения (по умолчанию 10) |

## Логи
Если прокси не запускается, ищи причину здесь:
`/data/adb/modules/tg_ws_proxy_magisk/proxy.log`
